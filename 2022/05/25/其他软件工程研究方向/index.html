基于随机块模型的多层次服务生态系统演化分析

本文基于随机块模型参数变化，从服务生态整体、服务社区以及服务个体三个层次对服务生态系统的演化进行全面的分析

背景

  • 多个 API 可以容易的组合成更大粒度的应用程序，称为服务聚合(mashup)，这些 mashup 可以为用户创造和提供各服务所无法提供 的独特新价值API 之间这种不断增长的“合作-竞争关系 ”催 生了“服务生态系统 ”，也称为 API 经济

  • 数据集：Programmable Web

  • 贡献：

    • 提出了一种利用带权重的度修正随机块模型对 ProgrammableWeb 服务生态系统进行建模的方法，有效地将服务生态系统中多层次信息统一到一个模型中
    • 从服务生态整体、服务社区以及服务个体三个层次对服务生态系统进行了演化分析
    • 提出了服务生态系统的整体演化点发现方法、服务社区的演化事件判别方法、服务个体演化阶段划分方法

  • 通过统计服务生态系统中例如密度、聚 类系数等网络性质来为服务推荐提供先验知识 .

Hierarchical Change Point Detection on Dynamic Networks

  • 其研究了社区演化的分类

本文研究了具有社区结构的网络上的变化点检测。它提出了一个框架，可以有效地检测网络中的局部和全局变化。重要的是，它可以清楚地区分两种类型的变化。

Main Idea

  • 网络的演化模式可以被分解为社区内的演化以及社区之间的演化
  • 社区间的交互可以通过一个超网络来近似，其中每个超节点都是一个契约社区

背景

  • 现有的变化点检测算法忽略网络的层次结构，因此很难区分社区内变化还是整体的变化
  • 如果一个社区明显大于其他社区，则该社区的重大变化将主导其他较小的社区。因此，无论网络的其余部分是否有任何变化，上述算法都会将其归类为变化点。也就是说，现有算法没有回答变化来自哪里，也无法回答变化是来自巨大的社区内部还是来自社区间的交互。

总结

  • 本文主要提出了一个框架，可以将网络先划分社区，再将社区视为一个超节点，随后分别研究超节点内以及超节点之间的变化

  • 这里将演化的检测视为变化点的检测

Quantifying social group evolution

本文定量的研究了社交网络演化的过程

数据集

  • 康奈尔大学图书馆电子打印 (cond-mat) 档案中的每月文章列表，跨越 142 个月，拥有超过 30,000 位作者

  • 某移动电话公司客户之间的通话记录，时间跨度为 52 周（超过 2 周累计），包含超过 400 万用户的通信模式

社区演化实例

  • 黄色表示旧的节点
  • 绿色表示新节点的数量
  • 放弃社区的成员为橙色或者紫色，具体取决于其为新节点还是老节点，仅加入一次就离开的为紫色

结论

  • 小集团的核心是一些牢固的关系，只要这些关系持续存在，他们周围的社区就很稳定。对于大型社区来说，维持这种策略似乎几乎是不可能的。我们的计算表明，大型社区的稳定条件是不断变化的，因此一段时间后几乎所有成员都被交换了。这种松散的、快速变化的社区让人想起机构，即使在所有成员都被新成员取代后，它也可以继续存在。例如，几年后，学校或公司的大多数成员可能会发生变化，但学校和公司在其存在期间的任何时候都可以作为一个不同的社区被发现。
